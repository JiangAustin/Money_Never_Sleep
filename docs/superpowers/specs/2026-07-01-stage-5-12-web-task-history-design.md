# 阶段 5.12：Web 任务历史视图设计

状态：已批准执行
日期：2026-07-01

## 背景

阶段 5.11 已让用户可以控制当前任务，但工作台仍然看不到近期任务列表。用户无法快速知道哪些任务失败、哪些任务被取消、哪些任务已完成，也无法从历史任务上下文进入报告。

## 目标

1. 在 Web 工作台展示最近任务列表。
2. 在真实 HTTP API 模式下读取 `GET /tasks?limit=`。
3. 让任务列表显示状态、股票、是否可重试，并与当前报告视图并存。

## 非目标

- 不做完整任务详情页。
- 不做批量操作。
- 不做分页或筛选。

## 设计

新增任务历史区域：

- 显示最近若干任务
- 展示 `task_id`、`symbol`、`status`
- 对失败/取消任务显示可重试提示

前端逻辑：

- 初始化时，如果有 `apiBaseUrl`，拉取 `GET /tasks?limit=10`
- 每次任务创建、取消、重试或轮询完成后刷新任务历史
- 离线模式下不展示真实任务历史，仅显示空态文案

## 验收

1. Web 测试覆盖任务历史区域和相关函数边界。
2. JS 语法检查通过。
3. 后端测试保持通过。
4. 文档更新任务历史入口说明。
