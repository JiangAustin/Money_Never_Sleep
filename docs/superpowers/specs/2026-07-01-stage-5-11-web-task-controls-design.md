# 阶段 5.11：Web 任务控制 UI 设计

状态：已批准执行
日期：2026-07-01

## 背景

阶段 5.8-5.10 已让服务端具备任务取消、重试、超时回收和 watchdog，但 Web 工作台仍然只能创建任务并被动轮询，用户无法从页面直接取消当前任务或重试失败任务。

## 目标

1. 在 Web 工作台展示当前任务 ID 和状态。
2. 在真实 HTTP API 模式下提供取消当前任务按钮。
3. 对失败任务提供重试入口。
4. 保持离线 mock 模式不受影响。

## 非目标

- 不做完整任务历史面板。
- 不做批量任务管理。
- 不改成前端框架。

## 设计

新增前端状态：

- `currentTaskId`
- `currentTaskState`
- `latestFailedTaskId`

新增 UI：

- 任务状态区域中的 `取消任务` 按钮
- 任务失败时显示 `重试任务` 按钮

新增前端函数：

- `cancelHttpAnalysisTask()`
- `retryHttpAnalysisTask()`

语义：

- cancel 仅在存在当前运行任务且有 `apiBaseUrl` 时可用。
- retry 仅在最近失败任务存在且有 `apiBaseUrl` 时可用。
- 成功重试后清空失败任务标记，进入新的轮询周期。

## 验收

1. Web workbench 测试覆盖任务控制按钮和函数边界。
2. JS 语法检查通过。
3. 后端测试无需新增端点改动时保持通过。
4. 文档更新任务控制入口说明。
